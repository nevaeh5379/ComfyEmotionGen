# 3. 통계 탭

통계 탭은 현재 세션의 작업 진행 상황과 워커 사용량을 한눈에 보여 줍니다. 진행 중인 큰 배치를 모니터링하거나, 끝난 뒤 회고할 때 사용합니다.

표시되는 모든 수치는 **헤더에서 선택한 세션**의 작업만 집계합니다. 전체 작업을 보려면 세션 선택기에서 "전체" 또는 다른 세션으로 바꾸세요.

## 3.1 접근 방법

통계 탭은 세 가지 방식으로 접근할 수 있습니다.

| 방식 | 설명 |
|---|---|
| **네비게이션 탭** | 상단 탭 바에서 "통계" 아이콘(BarChart3)을 클릭합니다 |
| **도킹 패널** | 창 모드(useWindowMode)에서 통계 패널을 화면 가장자리로 드래그하면 메인 패널에 결합됩니다 |
| **플로팅 창** | 창 모드에서 통계 패널을 독립 창으로 분리해 자유롭게 이동·크기 조정할 수 있습니다 |

> **참고**: 창 모드는 [설정 탭](./07-설정-및-단축키.md#창-모드)에서 `useWindowMode` 옵션으로 켜고 끕니다. 창 모드가 꺼져 있으면 모든 패널이 도킹된 기본 레이아웃으로 고정됩니다.

플로팅 창을 화면 가장자리(좌·우·상·하)로 드래그하면 스냅 결합 가이드가 표시됩니다. 마우스를 놓으면 "통계 패널이 메인 패널에 결합되었습니다"라는 알림이 뜨고 패널이 고정됩니다.

## 3.2 요약 카드

화면 맨 위에 핵심 지표가 가로로 늘어선 카드 행이 있습니다.

| 카드 | 의미 | 색상 |
|---|---|---|
| 총 작업 수 | 세션에 등록된 모든 작업 수 | 기본 |
| 성공률 | 완료된 작업의 비율(%) | 초록(text-ok) |
| 실패률 | 실패 또는 취소된 작업의 비율(%) | 빨강(text-bad) |
| 평균 실행 시간 | 완료된 작업의 평균 처리 시간 | 청록(text-info) |
| 재시도율 | 재시도(retryCount > 0)가 발생한 작업의 비율(%) | 노랑(text-warn) |

### 카드 인터랙션

- **호버 효과**: 마우스를 올리면 카드가 살짝 떠오르고(`-translate-y-0.5`) 색상에 맞는 글로우(광채) 효과가 나타납니다. 값은 5% 확대됩니다.
- **강화 애니메이션**: 값이 0인 카드는 `opacity-25`로 옅게 표시되어 시각적으로 가려집니다. 호버하면 `opacity-40`으로 약간 밝아집니다.
- **아이콘 애니메이션**: 평균 실행 시간 카드의 아이콘은 `animate-pulse`로 맥박 애니메이션이 적용되어 실시간 업데이트를 시각적으로 나타냅니다.
- **트렌드 표시**: `delta` 값이 있으면 ▲(증가) 또는 ▼(감소) 화살표와 함께 변화량이 표시됩니다.

### 카드 데이터 소스

모든 카드는 `useBackend()` 훅에서 제공하는 실시간 `jobs` 배열을 기반으로 계산됩니다. 작업 상태가 변경될 때마다 카드 값이 자동으로 갱신됩니다.

- **성공률** = 완료(done) 작업 수 / 총 작업 수 × 100
- **실패률** = 실패(error) + 취소(cancelled) 작업 수 / 총 작업 수 × 100
- **평균 실행 시간** = 완료된 작업의 `executionDurationMs` 평균 (분/초 형식, 예: `5m 30s`)
- **재시도율** = `retryCount > 0`인 작업 수 / 총 작업 수 × 100

## 3.3 상태 분포

도넛 차트(원형) 위젯이 각 상태가 차지하는 비율을 보여 줍니다.

- 색상은 상태별로 고정되어 있습니다 (대기·큐잉은 파랑 계열, 실행중은 청록, 완료는 초록, 실패는 빨강, 취소는 회색).
- 차트 위에 마우스를 올리면 상태명과 개수가 포함된 툴팁이 나옵니다 (예: "완료 42개").
- 차트 하단에 범례(레전드)가 표시되어 각 상태별 색상과 개수를 한눈에 볼 수 있습니다.
- 개수가 0인 상태는 차트에 표시되지 않습니다 (빈 상태 자동 제외).

### 상태 색상 매핑

| 상태 | 색상 코드 |
|---|---|
| 대기 (pending) | `#3b82f6` (파랑) |
| 큐잉 (queued) | `#60a5fa` (밝은 파랑) |
| 실행중 (running) | `#06b6d4` (청록) |
| 완료 (done) | `#10b981` (초록) |
| 실패 (error) | `#ef4444` (빨강) |
| 취소 (cancelled) | `#9ca3af` (회색) |

## 3.4 시간대별 처리 그래프

세션의 작업이 시간에 따라 어떻게 처리됐는지를 막대그래프로 보여 줍니다.

### 시간 범위 선택기

그래프 오른쪽 상단에 **"오늘 / 7일 / 전체"** 토글 버튼이 있습니다. 클릭하면 시간 범위를 즉시 변경할 수 있습니다.

| 범위 | 표시 단위 | 설명 |
|---|---|---|
| 오늘 | 시간별 (00:00 ~ 23:00) | 오늘 자정부터 현재까지 매 시간대별 작업 수 |
| 7일 | 일별 (MM/dd) | 최근 7일간의 일별 작업 수 |
| 전체 | 일별 (MM/dd) | 모든 기간의 작업 수 (날짜별 정렬) |

> **참고**: "오늘" 모드는 24개 시간대 버킷을 항상 표시합니다 (작업이 없는 시간대도 0으로 표시). "7일"과 "전체" 모드는 작업이 실제로 발생한 날짜만 표시합니다.

### 그래프 특징

- 가로축은 시각 또는 날짜, 세로축은 그 시간 구간에 들어간 작업 수입니다.
- 상태별로 색이 누적되어, 같은 시각에 완료·실패·취소가 얼마나 섞여 있었는지 한 번에 볼 수 있습니다.
- 막대 위에 마우스를 올리면 시간/날짜와 각 상태별 상세 개수가 포함된 커스텀 툴팁이 나타납니다.
- 큰 배치를 돌릴 때 워커 처리량의 안정성을 확인하는 데 유용합니다.

### 표시되는 상태

누적 막대그래프에는 완료(done), 실패(error), 취소(cancelled) 상태만 표시됩니다. 대기/큐잉/실행중 상태는 아직 종료되지 않은 작업이므로 시간대별 그래프에는 포함되지 않습니다.

## 3.5 워커별 성능

등록된 ComfyUI 워커별로 성능을 테이블로 표시합니다.

### 테이블 컬럼

| 컬럼 | 내용 |
|---|---|
| 워커 | 워커 ID(앞 8자리로 표시). 온라인이면 초록 점, 작업 중이면 파랑 점(맥박 애니메이션)이 표시됩니다 |
| 상태 | `offline`(회색), `busy`(청록), `idle`(초록) 중 하나의 뱃지로 표시 |
| 총 작업 수 | 해당 워커가 처리한 전체 작업 수 |
| 완료 | 정상적으로 완료한 작업 수 |
| 평균 시간 | 완료된 작업의 평균 처리 시간 (예: `2m 15s`, 완료 작업이 없으면 `N/A`) |
| 성공률 | 완료 비율(%). 80% 이상은 초록, 50~79%는 노랑, 50% 미만은 빨강으로 표시. 완료 작업이 없으면 `N/A` |

### 워커 상태 표시

- **초록 점** (`bg-ok`): 워커가 온라인(alive) 상태
- **파랑 점** (`bg-info`, `animate-pulse`): 워커가 현재 작업 처리 중(busy)
- 워커가 자주 오프라인으로 빠진다면 백엔드 로그를 확인하세요.
- 워커 추가·삭제는 [설정 탭](./07-설정-및-단축키.md#워커-관리)에서 합니다.

### 정렬

워커 테이블은 완료 수(`done`)가 많은 순으로 정렬되어 표시됩니다.

## 3.6 빈 상태

작업 데이터가 전혀 없을 때(총 작업 수 = 0) 화면 중앙에 "작업 데이터가 없습니다."라는 메시지가 표시됩니다.

## 3.7 실시간 업데이트

통계 탭의 모든 데이터는 백엔드와 실시간으로 동기화됩니다.

- `useBackend()` 훅이 WebSocket을 통해 백엔드에서 `jobs`와 `workers` 데이터를 실시간으로 수신합니다.
- 작업 상태가 변경되면 요약 카드, 차트, 워커 테이블이 자동으로 갱신됩니다.
- 매번 새로고침할 필요가 없습니다.

## 3.8 관련 설정

설정 탭의 다음 옵션들이 통계 탭과 관련이 있습니다.

| 설정 | 영향 |
|---|---|
| `useWindowMode` | 창 모드 활성화 시 통계 패널을 플로팅 창으로 분리하거나 도킹할 수 있습니다 |
| `progressCalculation` | 세션 진행률 계산 방식을 결정합니다 (`done` / `doneOrCancelled` / `doneOrFailed` / `excludeFromDenominator`) |

## 3.9 자주 묻는 것

**Q. 통계가 갑자기 적게 잡혀요.**
A. 세션이 새로 만들어지면 새 세션부터 다시 카운트됩니다. 헤더의 세션 선택기에서 이전 세션을 선택하면 그 세션의 통계가 보입니다.

**Q. 실행 중 작업이 0인데 큐가 줄지 않습니다.**
A. 큐가 일시중지된 상태일 수 있습니다. 헤더 오른쪽의 일시중지 버튼을 확인하세요. 워커가 모두 오프라인이어도 같은 증상이 납니다.

**Q. 차트가 비어 있어요.**
A. 새 세션이거나, 선택한 시간 범위에 종료된 작업이 없을 때 그렇습니다. 작업이 한 건이라도 끝나면(완료/실패/취소) 막대가 채워집니다.

**Q. 시간 범위를 어떻게 변경하나요?**
A. 시간대별 처리 그래프 오른쪽 상단의 "오늘 / 7일 / 전체" 버튼을 클릭하세요. 변경 즉시 그래프가 다시 렌더링됩니다.

**Q. 워커 테이블에서 성공률이 N/A로 나옵니다.**
A. 해당 워커가 완료된 작업이 없을 때 나타납니다. 아직 작업을 처리하지 않았거나, 모든 작업이 실패/취소 상태일 수 있습니다.

**Q. 통계 패널을 별도의 창으로 띄울 수 있나요?**
A. 설정 탭에서 창 모드(useWindowMode)를 활성화하면 통계 패널을 독립 창으로 분리할 수 있습니다. 분리된 창은 드래그로 이동하고, 화면 가장자리로 가져가면 메인 패널에 다시 결합됩니다.
