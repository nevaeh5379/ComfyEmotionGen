# 6. 템플릿 생성기 탭

템플릿 생성기는 **CEG 템플릿을 직접 손으로 안 쓰고, 변수만 채워 만드는** 도구입니다. 처음 사용하는 사용자가 CEG DSL을 익히기 전에 빠르게 첫 작업을 돌릴 수 있도록 만들어졌습니다.

## 6.1 개념

CEG 템플릿에는 다음과 같이 변수를 선언하는 부분이 자주 나옵니다.

```
{{set character = "1girl, blonde hair"}}
{{set emotion = "happy"}}
```

템플릿 생성기는 이 선언만 폼으로 떼어 내서 사용자에게 보여 줍니다. 사용자는 빈칸을 채우듯 값을 바꾸면 되고, 본문은 건드릴 필요가 없습니다.

## 6.2 화면 구성

화면은 세 영역으로 나뉩니다(데스크톱 기준 좌→우, 모바일에서는 탭으로 전환).

1. **왼쪽: 변수 폼** — 선택한 템플릿에 포함된 변수 목록과 입력 칸입니다.
2. **가운데: 코드 미리보기** — 변수 치환이 적용된 최종 CEG 코드입니다.
3. **오른쪽: 렌더 결과** — 이 템플릿이 만들어낼 작업 목록(파일명·프롬프트)입니다.

## 6.3 베이스 템플릿 선택

화면 상단의 셀렉트에서 베이스 템플릿을 고릅니다. 두 가지가 섞여 나옵니다.

- **저장됨(saved)** — 작업 탭에서 직접 저장한 사용자 템플릿입니다.
- **카테고리별 시스템 템플릿** — 백엔드의 `templates/` 폴더에 들어 있는 미리 만들어진 템플릿입니다. 캐릭터·감정·구도 같은 카테고리로 분류되어 있습니다.

선택한 템플릿에 변수가 있다면 왼쪽 폼이 채워집니다. 변수가 없는 템플릿이라면 폼은 비어 있고 코드 미리보기만 표시됩니다.

## 6.4 변수 편집

변수 폼에는 각 변수마다 두 개의 입력이 있습니다.

- **변수명** — 코드 안에서 사용할 식별자입니다. 보통 원래 이름 그대로 두면 됩니다.
- **값** — 큰따옴표 안에 들어갈 실제 문자열입니다. 여기에 원하는 캐릭터·감정·구도 등을 적습니다.

값을 입력하면 가운데 미리보기와 오른쪽 렌더 결과가 약간의 지연 뒤 자동 갱신됩니다. 본문 코드는 변수 선언부와 본문 안의 변수 사용 위치 모두에 안전하게 치환됩니다(문자열 리터럴 안은 건드리지 않습니다).

### 변수명을 바꾸려면

값뿐 아니라 변수명 자체도 바꿀 수 있습니다. 변수명 입력 칸의 값을 수정하면 다음에 모두 반영됩니다.

- 본문에서 해당 식별자가 쓰인 모든 위치
- 변수 선언부의 `{{set ...}}`

이 기능은 비슷한 템플릿을 여러 개 파생시킬 때 유용합니다(예: `character` → `character_a`, `character_b`).

## 6.5 렌더 결과 미리보기

오른쪽 패널에는 현재 코드가 만들어낼 작업이 표 형태로 표시됩니다.

- 각 행이 큐에 들어갈 한 건의 작업입니다.
- 위쪽 검색 입력으로 파일명·프롬프트 부분 일치 검색을 합니다.
- 파서 오류가 있으면 빨간 알림이 뜹니다. 변수 값에 깨진 따옴표가 있거나 본문 문법이 어긋났을 때 발생합니다.

## 6.6 저장하기

만든 템플릿이 마음에 들면 화면 상단의 **저장 이름** 칸에 이름을 적고 저장 버튼을 누릅니다.

- 기본 이름은 `<원본 이름> 커스텀` 형태로 채워집니다.
- 저장한 템플릿은 작업 탭의 CEG 에디터에서 **저장된 템플릿** 목록에 즉시 나타납니다.

## 6.7 작업 탭으로 이동

저장 후 **작업 탭으로 이동** 화살표 버튼을 누르면 새로 만든 템플릿이 활성 상태로 설정된 채 작업 탭이 열립니다. 그대로 실행 버튼만 누르면 큐가 채워집니다.

## 6.8 코드 복사 / 다운로드

코드 미리보기 패널 위쪽에는 다음 버튼이 있습니다.

- **복사** — 클립보드에 최종 코드를 복사합니다. 성공 시 잠시 ✓ 아이콘으로 바뀝니다.
- **다운로드** — `.ceg` 파일로 저장합니다. 다른 환경에서 다시 불러오거나 백업할 때 사용합니다.

## 6.9 자주 묻는 것

**Q. 저장된 템플릿 목록에 새로 만든 게 안 보입니다.**
A. 저장 이름이 비어 있으면 저장되지 않습니다. 다른 화면을 보고 와도 사용자 저장 템플릿은 보존되므로, 이름을 정해 다시 저장해 보세요.

**Q. 시스템 템플릿(카테고리) 목록이 비어 있어요.**
A. 백엔드 `templates/` 폴더가 비어 있거나 백엔드와 연결이 끊긴 상태일 수 있습니다. 헤더의 백엔드 상태등을 확인하세요.

**Q. 변수명을 바꿨더니 본문이 깨집니다.**
A. 변수명에 공백이나 한글, 특수문자가 포함되어 있는지 확인하세요. 식별자는 영문/숫자/언더스코어/하이픈만 허용합니다.

**Q. 코드 미리보기가 갱신되지 않습니다.**
A. 입력 후 잠시 지연이 있습니다. 그래도 갱신되지 않으면 파서 오류 메시지가 본문 위쪽에 표시되어 있는지 확인하세요.
