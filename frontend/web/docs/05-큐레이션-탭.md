# 5. 큐레이션 탭

큐레이션 탭은 **같은 조합으로 생성한 여러 후보 이미지 중에서 대표 이미지를 고르는** 전용 화면입니다.

갤러리 탭이 "낱장 이미지" 단위 작업이라면, 큐레이션 탭은 "한 조합당 후보 묶음" 단위 작업입니다. 배치 수를 3~5로 두고 같은 조합을 여러 번 돌린 뒤, 가장 잘 나온 한 장만 골라 데이터셋에 남기는 흐름에 최적화되어 있습니다.

## 5.1 화면 구성

- **상단 도구막대** — 축(그룹 기준) 선택, 뷰 모드, 옵션 토글 등이 있습니다.
- **좌측 사이드바** — 조합 그룹 목록과 미배정 항목 패널이 있습니다.
- **본문(그리드 / 상세 / 비교 / 토너먼트)** — 선택한 뷰 모드의 내용이 표시됩니다.

## 5.2 축(그룹 기준) 선택

도구막대의 **축** 셀렉트는 "후보를 어떻게 묶을지"를 결정합니다.

- **현재 템플릿** — 작업 탭에서 작성한 CEG 템플릿의 모든 축을 펼쳐 조합 단위로 묶습니다. 가장 일반적인 선택입니다.
- **저장된 템플릿** — 과거에 저장한 다른 템플릿을 기준으로 묶을 수도 있습니다.
- **자유 그룹화** — 템플릿 없이 파일명, 메타데이터 키 등 단순 기준으로 묶습니다(예: filename 기준, seed 제외).

축을 바꾸면 그룹 목록이 다시 계산됩니다. 계산이 끝나면 사이드바에 그룹이 채워집니다.

## 5.3 갤러리(그리드) 뷰

기본 뷰입니다. 조합 그룹의 대표 후보를 한 장씩 격자로 보여 줍니다.

- 그룹을 클릭하면 상세 뷰가 열려 그 그룹의 모든 후보를 펼쳐 봅니다.
- 썸네일 크기는 도구막대 또는 ctrl+휠로 조절합니다.
- 이미 대표가 지정된 그룹은 시각적으로 강조됩니다.

## 5.4 조합 상세 뷰

그리드의 그룹을 클릭하면 열리는 화면입니다. 한 조합의 모든 후보를 큰 화면에 줄지어 보여 줍니다.

각 후보 이미지 위에는 인덱스 번호(1, 2, 3 …)가 표시됩니다. 번호 키로 한 번에 대표를 지정할 수 있습니다.

| 키 | 동작 |
|---|---|
| `1` ~ `9` | 해당 번호의 후보를 **대표(승인)** 로 지정 |
| `↓` / `j` | 다음 조합으로 이동 |
| `↑` / `k` | 이전 조합으로 이동 |
| `r` / `R` | 이 조합을 다시 생성(시드만 다른 새 후보 추가) |
| `Esc` | 상세 뷰를 닫고 그리드로 복귀 |

후보 위에서 우클릭하면 통과/탈락/휴지통/태그 같은 갤러리와 동일한 메뉴가 뜹니다.

### 자동 진행

도구막대의 **자동 진행** 토글을 켜면 대표를 지정하자마자 다음 조합으로 자동 이동합니다. 빠르게 훑어 내려갈 때 매우 편리합니다.

### 자동 탈락 적용

설정의 **승인 시 나머지 자동 탈락**(autoApplyReject)이 켜져 있으면 한 후보를 승인할 때 나머지 후보는 자동으로 탈락 처리됩니다. 데이터셋에 대표만 남기고 싶을 때 켜 두세요.

## 5.5 비교 뷰

도구막대의 뷰 모드를 **비교**로 바꾸면 핀해 둔 이미지들을 나란히 봅니다.

- 갤러리 탭과 동일한 비교 핀 시스템을 공유합니다.
- 그리드 / 상세 뷰에서 `p` 키 또는 우클릭 메뉴로 핀합니다.
- 비교 뷰에서 가장 마음에 드는 한 장을 골라 대표로 승격시킬 수 있습니다.

## 5.6 토너먼트 뷰

후보가 많아 한 번에 보기 어려울 때 사용합니다. 두 장씩 짝지어 보여 주고 더 좋은 쪽을 고르면, 이긴 후보들끼리 다시 짝지어 마지막 한 장이 남을 때까지 반복합니다.

- 좌/우 클릭 또는 좌우 화살표로 승자를 선택합니다.
- 마지막 한 장이 자동으로 대표로 지정됩니다.
- 도중에 종료해도 그때까지의 평가는 후보별 점수에 반영됩니다.

## 5.7 미배정 항목 패널

축을 바꾸면 가끔 어느 그룹에도 속하지 않는 이미지가 생깁니다. 도구막대의 **미배정 보기** 토글로 패널을 열어 확인합니다.

- 미배정 이미지를 우클릭해 적절한 그룹으로 이동시킬 수 있습니다.
- 메타데이터가 빠져 있는 옛 이미지가 자주 여기에 모입니다.

## 5.8 일괄 동작

도구막대 우측에는 다음과 같은 일괄 도구가 있습니다.

- **일괄 재생성** — 현재 필터에 걸린 모든 그룹을 추가 생성합니다. 후보 수가 부족한 그룹에 한 장씩 더 보탤 때 유용합니다.
- **일괄 다운로드** — 현재 보이는 대표 이미지들을 ZIP으로 받습니다.
- **익스포트** — 데이터셋 익스포트입니다. 갤러리의 익스포트와 동일하지만, 큐레이션에서는 "대표만" 추리는 식의 필터가 들어가 있습니다.

각 동작은 진행률 토스트와 함께 백그라운드에서 진행됩니다.

## 5.9 작업 흐름 예시

대형 데이터셋을 만드는 일반적인 흐름입니다.

1. 작업 탭에서 CEG 템플릿을 작성하고 **배치 수**를 3~5로 두어 같은 조합을 여러 번 돌립니다.
2. 작업이 모두 끝나면 큐레이션 탭을 엽니다.
3. 축을 **현재 템플릿**으로 두면 조합별로 후보가 묶입니다.
4. 그룹을 클릭해 상세 뷰로 들어가고, `1~9` 키로 마음에 드는 한 장을 지정합니다. **자동 진행**이 켜져 있으면 다음 조합으로 바로 넘어갑니다.
5. 후보가 다 별로면 `r`로 추가 생성을 요청합니다. 작업 큐 매니저에 새 작업이 들어갑니다.
6. 모두 끝나면 익스포트로 데이터셋 ZIP을 받습니다.

## 5.10 단축키 (큐레이션)

| 키 | 동작 |
|---|---|
| `↓` / `j` | 다음 조합 |
| `↑` / `k` | 이전 조합 |
| `1` ~ `9` | 해당 인덱스 후보를 대표로 승인 |
| `r` / `R` | 조합 이미지 재생성 |
| `p` | 비교 핀 토글 |
| `Esc` | 조합 목록으로 복귀 |

전체 단축키는 화면 어디서나 `?` 키로 띄울 수 있습니다.
