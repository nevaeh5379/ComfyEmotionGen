# 2. 작업 탭

작업 탭은 가장 자주 머무는 화면입니다. 여기서 **작업 구성 패널**(왼쪽)과 **작업 큐 매니저**(오른쪽)가 나란히 표시됩니다.

- **작업 구성 패널** — 워크플로우와 CEG 템플릿을 작성하고, 큐를 만들어 실행하는 곳입니다.
- **작업 큐 매니저** — 큐에 들어간 작업의 진행 상황을 추적하고, 일시중지·재시도·취소 등을 합니다.

화면이 좁은 모바일에서는 두 패널이 한 번에 하나만 보이도록 하단 탭(에디터 / 상태 / 목록)으로 전환합니다.

---

## 2.1 워크플로우 작성

작업 구성 패널 상단의 탭에서 **워크플로우**를 선택합니다.

1. ComfyUI에서 워크플로우를 만들고 \"API 형식으로 저장\"한 JSON을 준비합니다.
2. 에디터에 붙여 넣습니다.
3. 상단의 이름 입력 칸에 이름을 적고 저장 버튼을 누르면 워크플로우 목록에 추가됩니다.
4. 같은 이름이 있으면 덮어쓸지, 새 이름으로 저장할지를 묻는 대화상자가 뜹니다.

저장한 워크플로우는 이름 입력 칸 오른쪽의 폴더 아이콘에서 다시 불러올 수 있습니다.

### 파일로 워크플로우 열기 (드래그 앤 드롭)

에디터 영역에 `.json` 파일을 드래그 앤 드롭하거나, 워크플로우 헤더의 폴더 아이콘 버튼을 클릭하여 파일을 선택하면 자동으로 유효성 검사 후 워크플로우 프리셋으로 저장됩니다. 파일 이름이 기존 프리셋과 겹치면 `(1)`, `(2)` 같은 접미사를 붙여 고유한 이름으로 저장됩니다.

### 워크플로우 JSON 복사 / 다운로드

워크플로우 헤더의 버튼으로 다음 작업을 할 수 있습니다.

- **복사** — 현재 워크플로우 JSON을 클립보드에 복사합니다.
- **다운로드** — `workflow.json` 파일로 저장합니다.

### 노드 개수 표시

워크플로우 JSON이 유효하게 파싱되면 헤더에 파싱된 노드 개수가 표시됩니다 (예: `15 Nodes`).

### 파싱 오류 표시

JSON 형식이 잘못되면 에디터 하단에 빨간색 경고 바가 나타나 오류 메시지를 표시합니다.

### 워크플로우 그래프 확인

작업 구성 패널 도구막대의 **그래프 보기** 버튼을 누르면 현재 워크플로우의 노드 그래프를 시각적으로 확인할 수 있습니다.

### 노드 매핑

워크플로우의 어느 노드 입력에 어떤 값을 끼울지 설정합니다.

- 워크플로우 에디터 아래의 **노드 매핑** 섹션을 펼칩니다.
- 각 노드의 입력(예: `text`, `seed`, `filename_prefix`) 옆에서 \"이 위치에 들어갈 값\"을 지정합니다. CEG 템플릿이 만들어내는 변수(예: `{{prompt}}`, `{{seed}}`, `{{filename}}`)와 연결합니다.

#### 매핑 소스 유형

각 매핑에서 다음 5가지 소스 유형을 선택할 수 있습니다.

| 소스 유형 | 설명 |
|---|---|
| **프롬프트** | CEG 템플릿의 `prompt` 필드 값을 해당 노드 입력에 주입합니다. |
| **파일명** | CEG 템플릿의 `filename` 필드 값을 해당 노드 입력에 주입합니다. |
| **시드** | 고정 시드 값을 입력하거나, 주사위 아이콘을 클릭하여 랜덤 시드로 전환합니다. |
| **이미지** | 로컬 이미지 파일을 선택하여 업로드합니다. 업로드가 완료되면 미리보기 썸네일이 표시됩니다. |
| **고정값** | 해당 노드 입력에 고정된 값을 설정합니다. 입력이 열거형(enum)이면 드롭다운에서 선택하고, 그렇지 않으면 직접 텍스트를 입력합니다. |

#### 자동 매핑

매핑 테이블 헤더의 마법봉 아이콘(✨)을 클릭하면 프롬프트·파일명·시드 매핑을 자동으로 설정합니다.

#### 프롬프트/파일명 매핑 경고

매핑이 하나 이상 설정되어 있지만 프롬프트 주입 매핑이나 파일명 주입 매핑이 누락된 경우, 노드 매핑 섹션 상단에 경고가 표시됩니다.

#### 매핑 추가

\"매핑 추가...\" 버튼을 클릭하면 아직 매핑되지 않은 노드 입력 목록이 드롭다운으로 나타납니다. 선택하면 해당 노드 입력에 대해 새로운 매핑 행이 추가되며, 입력 유형에 따라 적절한 기본 소스 유형(이미지 입력은 `image`, 수치 입력은 `seed`, 나머지는 `fixed`)이 자동 선택됩니다.

#### 매핑 프리셋 저장 / 불러오기

- 워크플로우가 저장된 상태여야 매핑 프리셋을 저장할 수 있습니다.
- 이름을 붙여 저장하면 같은 워크플로우에서 여러 매핑 조합을 관리할 수 있습니다.
- 저장된 프리셋은 폴더 아이콘에서 선택하여 불러오고, 업데이트하거나 삭제할 수 있습니다.

## 2.2 CEG 템플릿 작성

작업 구성 패널 상단의 탭에서 **CEG**를 선택합니다.

CEG 템플릿은 \"어떤 조합으로 몇 장을 만들지\"를 기술하는 DSL입니다. 예를 들어 캐릭터 × 감정 × 의상 같은 조합을 나열하면 모든 곱이 자동으로 풀려서 큐에 들어갑니다.

- 에디터에서 직접 작성하거나, 폴더 아이콘에서 저장된 템플릿을 불러옵니다.
- 변경 사항은 잠시 후 자동으로 미리보기에 반영됩니다. 문법 오류가 있으면 빨간 표시와 함께 메시지가 뜹니다.
- 처음이라면 [템플릿 생성기 탭](./06-템플릿-생성기.md)에서 변수만 채워 만드는 방식을 권장합니다.

### 파일로 템플릿 열기 (드래그 앤 드롭)

에디터 영역에 `.ceg` 파일을 드래그 앤 드롭하면 자동으로 템플릿 프리셋으로 저장됩니다.

### 템플릿 저장 / 불러오기 / 업데이트

- 상단 입력 칸에 이름을 적고 저장합니다.
- 이미 활성 템플릿이 있다면 **업데이트** 버튼이 나타나, 현재 편집 내용을 활성 템플릿에 덮어씁니다.
- `Ctrl+S`(Mac은 `Cmd+S`)로 빠르게 저장할 수 있습니다.
- 다운로드 버튼으로 `.ceg` 파일로 내보낼 수 있습니다.

### 미리보기 (파서 렌더링 결과)

도구막대의 **미리보기**(눈 아이콘) 버튼을 누르면 현재 템플릿이 만들어낼 작업 목록을 카드 형태로 봅니다.

- 각 카드가 실제로 실행될 한 건의 작업입니다.
- **파일명** — 렌더링된 파일명이 표시되며, 복사 버튼으로 클립보드에 복사할 수 있습니다.
- **메타데이터 뱃지** — 주입된 치환 변수(`emotion: happy`, `char: alice` 등)가 뱃지로 표시됩니다.
- **프롬프트** — 렌더링된 프롬프트가 표시되며, 마우스를 올리면 복사 버튼이 나타납니다.
- 축 필터로 제외된 항목은 회색/이탤릭체로 표시됩니다.

#### 미리보기 검색

미리보기 상단의 검색 바에서 다음 방식으로 검색할 수 있습니다.

- **일반 텍스트** — 파일명, 프롬프트, 메타데이터를 모두 검색합니다.
- `@` 접두사 — 파일명만 검색합니다 (예: `@alice`).
- `#` 접두사 — 프롬프트만 검색합니다 (예: `#smile`).
- `$` 접두사 — 메타데이터를 검색합니다. `$key:value` 형식으로 특정 변수의 값을 검색할 수 있습니다 (예: `$emotion:happy`).
- 입력 중인 텍스트에 대해 자동 완성 후보가 표시됩니다.
- \"초기화\" 버튼으로 검색 조건을 모두 지울 수 있습니다.

#### 미리보기 표 (선택 실행 모드)

선택 실행 모드에서 미리보기 표가 표시됩니다.

- 각 행은 FileName과 Prompt 두 열로 구성됩니다.
- 체크박스로 개별 항목을 선택/해제할 수 있습니다.
- 항목을 클릭하면 해당 작업의 상세 정보를 확인할 수 있습니다.

## 2.3 작업 실행

실행은 작업 구성 패널의 오른쪽 위 또는 헤더의 **실행** 버튼으로 합니다. 단축키는 `Ctrl+Enter`(Mac은 `Cmd+Enter`)입니다.

실행 버튼 옆의 ⋮ 메뉴에는 다음 옵션이 있습니다.

| 항목 | 설명 |
|---|---|
| 선택 실행 | 미리보기에서 체크한 일부만 큐에 추가합니다 |
| 랜덤 실행 | 전체에서 무작위로 N개만 골라 실행합니다 |
| 랜덤 개수 | 랜덤 실행에서 뽑을 개수입니다 |
| 축 필터 | 특정 축(예: emotion)의 일부 값만 실행되게 거릅니다 |
| 그래프 보기 | 워크플로우 그래프를 띄웁니다 |
| 배치 수 | 같은 조합을 N번 반복 실행합니다 (시드가 다른 여러 후보 만들기) |

예상 실행 수는 버튼 라벨에 표시됩니다. 필터를 걸면 그 수가 줄어듭니다.

### 축 필터

축 필터 시트에서 축 이름(예: `emotion`, `outfit`)별로 어떤 값을 포함할지 선택할 수 있습니다.

- 체크 해제한 값은 이번 실행에서 제외됩니다.
- 필터가 활성화된 동안에는 도구막대에 작은 표시가 뜹니다.
- 필터를 모두 해제하면 전체 조합이 다시 실행 대상이 됩니다.
- 축 필터가 적용되면 예상 실행 수가 줄어드는 것을 확인할 수 있습니다.

### 선택 실행

미리보기에서 항목을 체크하면 일부만 실행할 수 있습니다.

- 모두 선택 / 모두 해제 버튼이 있습니다.
- 실행 시점에 선택된 항목 수가 헤더에 표시됩니다.
- 선택 실행 시에도 배치 수(repeatCount) 설정이 적용됩니다.

### 템플릿 자동 파싱

CEG 템플릿을 작성하면 변경 사항이 잠시 후(디바운스) 자동으로 백엔드 파서 API에 전송되어 파싱됩니다. 파싱 오류가 발생하면 에디터 하단에 빨간색 오류 메시지가 표시됩니다.

## 2.4 작업 큐 매니저

오른쪽 패널은 큐에 들어간 작업의 진행 상황을 보여 줍니다.

### 통계 바

세션 안의 작업 상태별 개수를 막대로 보여 줍니다.

- **대기 / 큐 / 진행 / 완료 / 실패 / 취소** 의 6단계 상태입니다.
- 막대를 클릭하면 그 상태로 표 필터가 걸립니다.
- 하단에 **세션 전체 진행률** 바가 표시되며, 완료된 작업 수와 비율을百分比로 보여 줍니다. 진행률 계산 방식은 [설정](./07-설정-및-단축키.md)에서 변경할 수 있습니다 (완료 기준, 완료+취소 기준, 완료+실패 기준, 실패/취소 제외 기준).

### 실행 중 작업 배너

지금 실행 중인 작업이 있을 때 상단에 표시됩니다.

- 파일명, 현재 처리 중인 노드 이름, 진행률 퍼센트가 표시됩니다.
- **예상 완료 시간(ETA)** 이 표시됩니다.
- 전체 노드 진행률(완료된 노드 수 / 전체 노드 수)도 별도로 표시됩니다.
- 여러 작업이 동시에 실행 중이면 각 작업마다 개별 배너가 표시됩니다.

### 작업 표

큐에 있는 모든 작업이 표로 나옵니다.

- **상태 표시** — 각 행의 왼쪽 가장자리와 상태 필드에 색상이 표시됩니다 (완료: 초록, 실패: 빨강, 진행: 파랑 점멸, 큐: 노랑, 대기: 회색).
- **파일명 색상 점** — 파일명 앞에 파일명 해시로 생성된 고유 색상의 점이 표시됩니다.
- **정렬** — 컬럼 헤더의 화살표로 `상태`, `파일명`, `생성`, `소요` 기준으로 정렬합니다.
- **행 클릭** — 상세 시트가 열려 프롬프트, 노드 매핑, 결과 이미지, 메타데이터를 확인합니다.
- **완료 작업 호버** — 완료된 작업 행에 마우스를 올리면 생성된 이미지 썸네일(최대 6개)이 호버 카드 형태로 표시됩니다.
- **상태 필터 탭** — `전체`, `활성`(대기+큐+진행), `완료`, `실패`(에러+취소) 탭으로 빠르게 필터링합니다. 각 탭에 작업 수가 표시됩니다.
- **검색** — 파일명·프롬프트 부분 일치로 좁힙니다.

#### 페이지네이션

작업 수가 페이지 크기(기본 50개)를 초과하면 하단에 페이지네이션 컨트롤이 표시됩니다.

#### 모바일 카드 뷰

모바일 화면에서는 표 대신 카드 형태의 목록으로 표시됩니다. 각 카드에 상태 색상 표시, 파일명, 생성 시간, 소요 시간이 표시됩니다.

### 고급 검색 필터

작업 표 도구막대의 필터 아이콘(🔍)을 클릭하면 고급 검색 패널이 열립니다.

- **태그 기반 검색** — 텍스트를 입력하고 엔터를 누르면 검색 태그가 추가됩니다. 여러 태그는 AND 조건으로 필터링됩니다.
- **검색 구문**
  - 일반 텍스트: 파일명, 프롬프트, 에러, 메타데이터를 모두 검색
  - `@` 접두사: 파일명만 검색 (예: `@alice`)
  - `#` 접두사: 프롬프트만 검색 (예: `#smile`)
  - `$` 접두사: 에러 메시지 검색 (예: `$timeout`)
- **자동 완성** — 입력 중인 텍스트에 대해 파일명 토큰, 프롬프트 토큰, 에러 토큰에서 자동 완성 후보가 표시됩니다.
- **필터 초기화** — 모든 검색 태그와 기간 필터를 한 번에 지울 수 있습니다.
- 필터가 활성화되면 필터 아이콘에 점멸 표시가 나타납니다.

### 기간 필터

작업 표 도구막대에 시작일/종료일 선택기가 있습니다.

- **빠른 선택 버튼** — `1h`(최근 1시간), `오늘`, `24h`(최근 24시간) 버튼으로 빠르게 설정합니다.
- 기간이 설정되면 도구막대에 초록색 점멸 표시가 나타납니다.
- 기간 필터는 검색 태그 필터와 함께 사용할 수 있습니다.

### 실패 항목 대량 삭제

`실패` 탭에서 실패/취소된 작업을 대량으로 삭제할 수 있습니다.

- **전체 선택** — 현재 표시된 실패/취소 작업 전체를 선택합니다.
- **선택 해제** — 모든 선택을 해제합니다.
- **삭제** — 선택한 작업을 영구 삭제합니다. 확인 대화상자가 한 번 뜹니다.

### 전체 일시중지 / 재개

헤더 오른쪽의 **일시중지** 버튼으로 큐 전체를 잠시 멈출 수 있습니다. 이미 워커에서 실행 중인 작업은 끝까지 진행되고, 큐에 대기 중인 작업이 워커로 내려가지 않게 됩니다. 다시 누르면 재개됩니다.

### 전체 취소

헤더의 ⋮ 메뉴 또는 큐 매니저 도구막대의 **모두 취소**를 누르면 진행 중·대기 중인 작업을 모두 취소합니다. 확인 대화상자가 한 번 뜹니다.

### 실패 작업 재시도 / 삭제

큐 매니저의 ⋮ 메뉴에서 다음을 할 수 있습니다.

- **실패 항목 모두 재시도** — 상태가 실패·취소인 작업을 다시 큐에 넣습니다.
- **실패 항목 모두 삭제** — 실패·취소된 작업 기록을 영구 삭제합니다. 확인 대화상자가 한 번 뜹니다.

## 2.5 작업 상세 시트

작업 표에서 행을 클릭하면 오른쪽으로 슬라이드되는 상세 시트가 열립니다.

### 기본 정보

- **상태 뱃지** — 현재 작업 상태가 표시됩니다.
- **작업 ID** — 단축 ID가 표시됩니다.
- **파일명** — 렌더링된 파일명이 표시됩니다.

### 프롬프트

- 렌더링된 프롬프트 전체가 표시됩니다.
- 우측 상단의 복사 버튼으로 클립보드에 복사할 수 있습니다.

### 에러 로그

- 작업이 실패한 경우, 에러 메시지가 빨간색 영역에 표시됩니다.
- 복사 버튼으로 에러 메시지를 복사할 수 있습니다.

### 진행 타임라인

시각적인 타임라인으로 작업의 생명주기를 추적합니다.

- **작업 생성됨** — 작업이 큐에 들어온 시각
- **렌더링 시작** — 워커가 작업을 시작한 시각 (진행 중이면 점멸 표시)
- **렌더링 완료 / 실패 / 취소** — 최종 상태와 시각
- **총 소요 시간** — 렌더링에 걸린 총 시간 (초/분 단위)
- 아직 도달하지 않은 단계는 흐리게 표시됩니다.

### 액션 버튼

작업 상태에 따라 다른 버튼이 표시됩니다.

- **대기/큐/진행 중** — 취소 버튼
- **실패/취소** — 재시도 버튼, 삭제 버튼

### 생성 이미지

완료된 작업의 생성된 이미지가 그리드 형태로 표시됩니다.

- 이미지를 클릭하면 라이트박스 뷰어가 열립니다.
- 여러 이미지가 있으면 좌우 네비게이션 버튼과 하단 썸네일 스트립으로 이미지를 전환할 수 있습니다.
- 현재 이미지 번호와 전체 수가 표시됩니다.

## 2.6 세션 관리

세션은 작업 큐를 시간 단위로 묶어 주는 마커입니다.

### 새 세션 만들기

헤더 가운데의 세션 선택기를 열고 **새 세션** 버튼을 누릅니다.

- 비어 있는 이전 세션은 자동으로 정리됩니다.
- 새 세션이 활성으로 지정되어 다음 작업부터 새 세션에 들어갑니다.

### 세션 전환

세션 선택기의 목록에서 다른 세션을 선택하면 작업 큐 매니저와 통계 탭이 그 세션의 작업만 보여 줍니다.

- 가장 최근에 생성된 세션이 기본 선택입니다.
- 각 세션 옆에 작업 수가 표시됩니다.
- 활성 세션에는 초록색 점멸 표시가 나타납니다.

### 배치 완료 알림

활성 작업이 모두 끝나면 토스트로 결과 요약(완료 / 실패 / 취소 수)이 뜹니다. 웹훅이 켜져 있다면 같은 내용이 외부로도 전송됩니다.

## 2.7 패널 분리와 도킹

데스크톱에서 작업 구성 패널과 작업 큐 매니저는 별도의 창으로 분리할 수 있습니다(설정에서 창 모드가 켜져 있어야 합니다).

- 패널 헤더의 ↗ 아이콘으로 분리합니다.
- 패널 헤더 영역을 드래그해서 화면 가장자리에 가져다 놓으면 자동으로 도킹됩니다.
- 통계 / 갤러리 / 큐레이션 탭도 같은 방식으로 작업 탭 옆에 도킹할 수 있습니다. 헤더의 해당 탭 이름을 드래그하세요.
- **레이아웃 방향 전환** — 작업 구성 패널 헤더의 레이아웃 전환 버튼(가로/세로 아이콘)으로 두 패널의 배치를 가로 분할(왼쪽/오른쪽)과 세로 분할(위/아래)로 전환할 수 있습니다.

도킹 위치(상하 좌우)에 따라 작업 패널들의 배치가 가로/세로로 바뀝니다.
