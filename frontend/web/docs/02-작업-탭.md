# 2. 작업 탭

작업 탭은 가장 자주 머무는 화면입니다. 여기서 **작업 구성 패널**(왼쪽)과 **작업 큐 매니저**(오른쪽)가 나란히 표시됩니다.

- **작업 구성 패널** — 워크플로우와 CEG 템플릿을 작성하고, 큐를 만들어 실행하는 곳입니다.
- **작업 큐 매니저** — 큐에 들어간 작업의 진행 상황을 추적하고, 일시중지·재시도·취소 등을 합니다.

화면이 좁은 모바일에서는 두 패널이 한 번에 하나만 보이도록 하단 탭(에디터 / 상태 / 목록)으로 전환합니다.

---

## 2.1 워크플로우 작성

작업 구성 패널 상단의 탭에서 **워크플로우**를 선택합니다.

1. ComfyUI에서 워크플로우를 만들고 "API 형식으로 저장"한 JSON을 준비합니다.
2. 에디터에 붙여 넣습니다.
3. 상단의 이름 입력 칸에 이름을 적고 저장 버튼을 누르면 워크플로우 목록에 추가됩니다.
4. 같은 이름이 있으면 덮어쓸지, 새 이름으로 저장할지를 묻는 대화상자가 뜹니다.

저장한 워크플로우는 이름 입력 칸 오른쪽의 폴더 아이콘에서 다시 불러올 수 있습니다.

### 워크플로우 그래프 확인

작업 구성 패널 도구막대의 **그래프 보기** 버튼을 누르면 현재 워크플로우의 노드 그래프를 시각적으로 확인할 수 있습니다.

### 노드 매핑

워크플로우의 어느 노드 입력에 어떤 값을 끼울지 설정합니다.

- 워크플로우 에디터 아래의 **노드 매핑** 섹션을 펼칩니다.
- 각 노드의 입력(예: `text`, `seed`, `filename_prefix`) 옆에서 "이 위치에 들어갈 값"을 지정합니다. CEG 템플릿이 만들어내는 변수(예: `{{prompt}}`, `{{seed}}`, `{{filename}}`)와 연결합니다.
- 자주 쓰는 매핑은 **프리셋**으로 저장해 두면 다음 워크플로우에서 한 번에 불러올 수 있습니다.

---

## 2.2 CEG 템플릿 작성

작업 구성 패널 상단의 탭에서 **CEG**를 선택합니다.

CEG 템플릿은 "어떤 조합으로 몇 장을 만들지"를 기술하는 DSL입니다. 예를 들어 캐릭터 × 감정 × 의상 같은 조합을 나열하면 모든 곱이 자동으로 풀려서 큐에 들어갑니다.

- 에디터에서 직접 작성하거나, 폴더 아이콘에서 저장된 템플릿을 불러옵니다.
- 변경 사항은 잠시 후 자동으로 미리보기에 반영됩니다. 문법 오류가 있으면 빨간 표시와 함께 메시지가 뜹니다.
- 처음이라면 [템플릿 생성기 탭](./06-템플릿-생성기.md)에서 변수만 채워 만드는 방식을 권장합니다.

### 템플릿 저장 / 불러오기 / 업데이트

- 상단 입력 칸에 이름을 적고 저장합니다.
- 이미 활성 템플릿이 있다면 **업데이트** 버튼이 나타나, 현재 편집 내용을 활성 템플릿에 덮어씁니다.
- `Ctrl+S`(Mac은 `Cmd+S`)로 빠르게 저장할 수 있습니다.
- 다운로드 버튼으로 `.ceg` 파일로 내보낼 수 있습니다.

### 미리보기

도구막대의 **미리보기**(눈 아이콘) 버튼을 누르면 현재 템플릿이 만들어낼 작업 목록을 표로 봅니다.

- 각 행이 실제로 실행될 한 건의 작업입니다.
- 상단 검색 칸으로 filename·prompt를 부분 일치 검색할 수 있습니다.
- 예상 작업 수가 너무 많거나 적으면 템플릿이나 필터를 다시 확인하세요.

## 2.3 작업 실행

실행은 작업 구성 패널의 오른쪽 위 또는 헤더의 **실행** 버튼으로 합니다. 단축키는 `Ctrl+Enter`(Mac은 `Cmd+Enter`)입니다.

실행 버튼 옆의 ⋮ 메뉴에는 다음 옵션이 있습니다.

| 항목 | 설명 |
|---|---|
| 선택 실행 | 미리보기에서 체크한 일부만 큐에 추가합니다 |
| 랜덤 실행 | 전체에서 무작위로 N개만 골라 실행합니다 |
| 랜덤 개수 | 랜덤 실행에서 뽑을 개수입니다 |
| 축 필터 | 특정 축(예: emotion)의 일부 값만 실행되게 거릅니다 |
| 그래프 보기 | 워크플로우 그래프를 띄웁니다 |
| 배치 수 | 같은 조합을 N번 반복 실행합니다 (시드가 다른 여러 후보 만들기) |

예상 실행 수는 버튼 라벨에 표시됩니다. 필터를 걸면 그 수가 줄어듭니다.

### 축 필터

축 필터 시트에서 축 이름(예: `emotion`, `outfit`)별로 어떤 값을 포함할지 선택할 수 있습니다.

- 체크 해제한 값은 이번 실행에서 제외됩니다.
- 필터가 활성화된 동안에는 도구막대에 작은 표시가 뜹니다.
- 필터를 모두 해제하면 전체 조합이 다시 실행 대상이 됩니다.

### 선택 실행

미리보기에서 항목을 체크하면 일부만 실행할 수 있습니다.

- 모두 선택 / 모두 해제 버튼이 있습니다.
- 실행 시점에 선택된 항목 수가 헤더에 표시됩니다.

---

## 2.4 작업 큐 매니저

오른쪽 패널은 큐에 들어간 작업의 진행 상황을 보여 줍니다.

### 통계 바

세션 안의 작업 상태별 개수를 막대로 보여 줍니다.

- **대기 / 큐잉 / 실행중 / 완료 / 실패 / 취소** 의 6단계 상태입니다.
- 막대를 클릭하면 그 상태로 표 필터가 걸립니다.

### 실행 중 작업 배너

지금 실행 중인 작업이 있을 때 상단에 표시됩니다. 진행률, 워커, 경과 시간이 보이고, 작업을 클릭하면 상세 시트로 이동합니다.

### 작업 표

큐에 있는 모든 작업이 표로 나옵니다.

- 정렬: 컬럼 헤더의 화살표로 시간 / 상태 / 이름 등을 정렬합니다.
- 행 클릭: 상세 시트가 열려 프롬프트, 노드 매핑, 결과 이미지, 메타데이터를 확인합니다.
- 상태 필터: 위쪽 통계 바 또는 표 도구막대의 셀렉트에서 특정 상태만 봅니다.
- 검색: 파일명·프롬프트 부분 일치로 좁힙니다.

### 전체 일시중지 / 재개

헤더 오른쪽의 **일시중지** 버튼으로 큐 전체를 잠시 멈출 수 있습니다. 이미 워커에서 실행 중인 작업은 끝까지 진행되고, 큐에 대기 중인 작업이 워커로 내려가지 않게 됩니다. 다시 누르면 재개됩니다.

### 전체 취소

헤더의 ⋮ 메뉴 또는 큐 매니저 도구막대의 **모두 취소**를 누르면 진행 중·대기 중인 작업을 모두 취소합니다. 확인 대화상자가 한 번 뜹니다.

### 실패 작업 재시도 / 삭제

큐 매니저의 ⋮ 메뉴에서 다음을 할 수 있습니다.

- **실패 항목 모두 재시도** — 상태가 실패·취소인 작업을 다시 큐에 넣습니다.
- **실패 항목 모두 삭제** — 실패·취소된 작업 기록을 영구 삭제합니다. 확인 대화상자가 한 번 뜹니다.

---

## 2.5 세션 관리

세션은 작업 큐를 시간 단위로 묶어 주는 마커입니다.

### 새 세션 만들기

헤더 가운데의 세션 선택기를 열고 **새 세션** 버튼을 누릅니다.

- 비어 있는 이전 세션은 자동으로 정리됩니다.
- 새 세션이 활성으로 지정되어 다음 작업부터 새 세션에 들어갑니다.

### 세션 전환

세션 선택기의 목록에서 다른 세션을 선택하면 작업 큐 매니저와 통계 탭이 그 세션의 작업만 보여 줍니다.

- 가장 최근에 생성된 세션이 기본 선택입니다.
- 각 세션 옆에 작업 수가 표시됩니다.

### 배치 완료 알림

활성 작업이 모두 끝나면 토스트로 결과 요약(완료 / 실패 / 취소 수)이 뜹니다. 웹훅이 켜져 있다면 같은 내용이 외부로도 전송됩니다.

---

## 2.6 패널 분리와 도킹

데스크톱에서 작업 구성 패널과 작업 큐 매니저는 별도의 창으로 분리할 수 있습니다(설정에서 창 모드가 켜져 있어야 합니다).

- 패널 헤더의 ↗ 아이콘으로 분리합니다.
- 패널 헤더 영역을 드래그해서 화면 가장자리에 가져다 놓으면 자동으로 도킹됩니다.
- 통계 / 갤러리 / 큐레이션 탭도 같은 방식으로 작업 탭 옆에 도킹할 수 있습니다. 헤더의 해당 탭 이름을 드래그하세요.

도킹 위치(상하 좌우)에 따라 작업 패널들의 배치가 가로/세로로 바뀝니다.

## 2.7 단축키 (작업 탭)

| 기능 | 단축키 |
|---|---|
| 큐 생성 및 실행 | `Ctrl+Enter` / `Cmd+Enter` |
| 에디터 빠른 저장 | `Ctrl+S` / `Cmd+S` |
| 단축키 안내 패널 열기 | `?` |
| 탭 전환 | `Alt+1..5` 또는 `Ctrl+Shift+1..5` |

전체 단축키 목록은 [설정 및 단축키](./07-설정-및-단축키.md) 문서에 있습니다.
