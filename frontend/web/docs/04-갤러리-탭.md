# 4. 갤러리 탭

갤러리 탭은 지금까지 생성하고 저장한 모든 이미지를 탐색·정리하는 공간입니다. 검색, 태그, 폴더 분류, 큐레이션 상태(통과/탈락/휴지통), 비교 뷰, 데이터셋 익스포트까지 한 곳에서 처리합니다.

갤러리 패널은 메인 화면에 고정(도킹)되거나, 독립적인 부동 창으로 떼어내어 사용 가능합니다. 부동 창은 화면 가장자리로 드래그하면 자동으로 스냅 결합됩니다.

## 4.1 화면 구성

상단에 **갤러리 도구막대**가 있고, 그 아래로 이미지 그리드가 펼쳐집니다. 도구막대는 화면이 좁아지면 자동으로 축소되어 가장 자주 쓰는 것만 보이게 합니다.

- **검색 입력** — 파일명·태그·메타데이터 검색 칸이 들어 있습니다.
- **상태 필터** — 통과 / 탈락 / 휴지통 / 전체 등 큐레이션 상태로 좁힙니다.
- **뷰 모드** — 그리드 / 그룹 / 비교 모드 전환입니다.
- **정렬** — 날짜 / 파일명 / 크기 기준, 오름차순/내림차순 토글입니다.
- **썸네일 크기 조절** — 슬라이더로 120px~320px까지 조절 (기본값 180px). 설정은 로컬 스토리지에 자동 저장됩니다.
- **필터 표시/숨기기** — 필터 버튼을 누르면 확장된 검색 패널(파일명, 태그, 메타데이터 필터 + 리젝 숨기기 토글)이 나타납니다. 필터가 적용되면 필터 아이콘에 파란색 점(핑 애니메이션)이 표시됩니다.
- **더보기 메뉴** — 새로고침, 휴지통 비우기 등이 들어 있습니다.

## 4.2 이미지 둘러보기

### 그리드 뷰

기본 뷰입니다. 썸네일이 격자로 나열됩니다.

- **클릭**: 이미지가 큰 화면(상세 뷰어)으로 열립니다.
- **마우스 위에 올림**: 설정에서 \"호버 미리보기\"가 켜져 있으면 큰 미리보기가 즉시 표시됩니다. 그리드 카드의 이미지는 호버 시 약간 확대(`scale-105`)되는 애니메이션이 적용됩니다.
- **우클릭**: 컨텍스트 메뉴가 열립니다.

#### 그리드 카드 표시 정보

각 카드에는 다음 정보가 표시됩니다:

- **체크박스 버튼 (왼쪽 위)**: 선택 모드에서 선택 상태 표시. 핀 고정된 이미지는 핀 아이콘으로 표시됩니다.
- **상태 라벨 (왼쪽 위)**: 데스크톱에서만 표시되는 큐레이션 상태 배지 (대기/통과/탈락/휴지통).
- **휴지통 버튼 (오른쪽 위)**: 데스크톱에서만 보이는 빠른 휴지통/복원 버튼.
- **태그 배지 (아래쪽)**: 데스크톱에서만 표시되는 이미지 태그 목록.

#### 컨텍스트 메뉴 (우클릭)

이미지 카드에서 우클릭하면 다음 메뉴가 표시됩니다:

- **비교에 추가 / 비교에서 제거**: 비교 뷰에 핀 고정 토글
- **통과**: 큐레이션 상태를 통과(approved)로 변경
- **탈락**: 큐레이션 상태를 탈락(rejected)으로 변경
- **대기**: 큐레이션 상태를 대기(pending)로 변경
- **휴지통 / 복원**: 휴지통으로 이동 또는 휴지통에서 복원
- **상세 보기**: 상세 뷰어 열기
- **워크플로우 JSON 복사**: 이미지에 연결된 ComfyUI 워크플로우 JSON을 클립보드에 복사
- **태그 전체 복사**: 이미지의 모든 태그를 쉼표 구분 텍스트로 복사
- **원래 이름으로 다운로드**: 원본 파일명으로 이미지 다운로드
- **이미지 URL 복사**: 이미지 접근 URL을 클립보드에 복사

#### 로딩 및 에러 처리

- **로딩 중**: 스켈레톤 플레이스홀더(12개, 높이 랜덤)로 로딩 상태를 표시합니다.
- **이미지 로딩 실패**: 깨진 이미지는 `ImageOff` 아이콘으로 대체 표시됩니다.

### 그룹 뷰

같은 파일명 베이스나 메타데이터 키로 묶어 봅니다. 같은 조합의 여러 시드 결과를 한 묶음으로 보고 싶을 때 유용합니다.

각 그룹에는 다음 정보가 표시됩니다:

- **그룹 이름** (모노스페이스 폰트)
- **통계**: 총 개수, 통과 수, 탈락 수, 휴지통 수
- **접기/펴기 토글**: 개별 그룹을 접거나 펼칠 수 있습니다.
- **재생성 버튼**: 해당 그룹에 새로운 이미지를 추가 생성할 수 있습니다. 생성 수량을 입력하는 다이얼로그가 열립니다 (1~100장).
- **모두 접기/펴기**: 모든 그룹을 한 번에 접거나 펼칩니다.

그룹 모드도 페이지네이션을 지원합니다 (페이지당 20개 그룹, 현재 페이지 ±1 표시 + 생략부호).

### 비교 뷰

여러 이미지를 한 화면에 나란히 띄워 차이점을 비교합니다.

- 그리드에서 이미지에 마우스를 올리고 `p` 키를 누르거나, 우클릭 → **비교 핀 추가**로 비교 목록에 넣습니다.
- 도구막대의 뷰 모드를 **비교**로 바꾸면 핀된 이미지들이 동시에 표시됩니다.
- 비교 뷰에서 다시 `p`를 누르면 핀이 해제됩니다.
- 핀된 이미지는 1장일 때 1열, 2장일 때 2열, 3장 이상일 때 3열로 나뉩니다.
- 비교 뷰의 각 이미지는 마그니파이어(돋보기) 호버 미리보기를 지원합니다.
- 핀 제거 버튼(파란색 원형)으로 개별 이미지를 비교에서 제거할 수 있습니다.
- 비교 뷰에서도 우클릭 컨텍스트 메뉴로 상태 변경, URL 복사 등을 할 수 있습니다.
- 핀된 이미지가 없을 때 비교 모드로 전환하면 안내 메시지가 표시됩니다.

## 4.3 검색 / 필터

### 태그 기반 검색 (TagInputSearch)

도구막대의 검색 입력은 태그 기반 검색 컴포넌트를 사용합니다. 입력 시 자동 완성 드롭다운이 나타나며, 다음 타입의 후보를 구분 표시합니다:

- **파일명** (파란색 배지): `@` 접두사로 추가
- **태그/프롬프트** (초록색 배지): `#` 접두사로 추가
- **메타데이터/에러** (빨간색 배지): `$` 접두사로 추가

자동 완성 드롭다운의 특징:

- 입력한 텍스트와 일치하는 문자가 **굵게 하이라이트**됩니다.
- `↑`/`↓` 키로 후보를 이동하고, `Enter`로 선택합니다.
- `Esc`로 드롭다운 닫기.
- `Backspace`로 마지막 태그 제거 (입력칸이 비어있을 때).
- 추가된 태그는 배지로 표시되며, `×` 버튼으로 개별 제거할 수 있습니다.

### 필터 패널

필터 버튼(🔍)을 누르면 확장된 검색 패널이 나타납니다:

1. **파일명 필터** — 파일명 부분 일치 검색
2. **태그 필터** — 태그 부분 일치 검색
3. **메타데이터/prompt 검색** — 프롬프트 텍스트 전체에서 부분 일치 검색
4. **리젝 숨기기** — 탈락(rejected) 상태 이미지를 숨김 처리

**필터 초기화** 버튼으로 모든 필터를 한 번에 비웁니다.

### 상태 필터

도구막대의 상태 셀렉트에서 다음 중 하나를 선택할 수 있습니다.

- **전체(all)** — 모든 상태의 이미지
- **대기(pending)** — 아직 분류하지 않은 이미지
- **통과(approved)** — `1` 키로 통과 처리한 이미지
- **탈락(rejected)** — `2` 키로 탈락 처리한 이미지
- **휴지통(trashed)** — `3` 키로 휴지통에 보낸 이미지

### 정렬

도구막대의 정렬 셀렉트에서 다음 기준 중 하나를 선택합니다:

- **날짜순** (`createdAt`): 생성 시간 기준 (기본값, 내림차순)
- **파일명순** (`filename`): 원본 파일명 기준
- **크기순** (`sizeBytes`): 파일 크기 기준

정렬 방향(오름차순/내림차순)은 화살표 버튼을 클릭하거나, 같은 기준을 다시 선택하면 토글됩니다.

## 4.4 브레드크럼 태그 폴더 탐색

갤러리는 Danbooru 스타일의 **태그 기반 가상 폴더 시스템**을 제공합니다.

- 그리드 상단에 브레드크럼 네비게이션 바가 표시됩니다.
- **Home** 링크로 루트 폴더로 이동합니다.
- 현재 필터 조건에 맞는 **하위 토큰(태그)**이 폴더 버튼으로 표시됩니다. 각 버튼에는 해당 태그를 가진 이미지 수(count)가 표시됩니다.
- 하위 폴더 버튼을 클릭하면 해당 태그로 추가 필터링됩니다.
- **폴더 검색** 입력창으로 하위 태그를 빠르게 찾을 수 있습니다.
- 최대 30개의 하위 폴더가 표시되며, 스크롤로 나머지를 볼 수 있습니다.

### 브레드크럼 우클릭 메뉴

- 브레드크럼 태그를 우클릭하면:
  - **이 위치까지 경로 자르기**: 해당 태그 이후의 경로를 제거
  - **경로에서 제거**: 해당 태그만 제거
  - **태그명 복사**: 태그 이름을 클립보드에 복사
- 하위 폴더 버튼을 우클릭하면:
  - **맨 앞에 경로 추가**: 해당 태그를 브레드크럼 맨 앞에 추가
  - **태그명 복사**
- 빈 영역을 우클릭하면:
  - **상위 폴더로 이동**
  - **홈 폴더로 이동**
  - **하위 폴더로 이동** (서브 메뉴로 최대 40개 표시)
  - **경로 새로고침**

## 4.5 큐레이션 상태 바꾸기

이미지를 빠르게 분류하려면 키보드 단축키를 사용합니다.

| 키 | 동작 |
|---|---|
| `1` | 포커스된 이미지를 **통과**로 표시 |
| `2` | 포커스된 이미지를 **탈락**으로 표시 |
| `3` | 휴지통으로 이동 (휴지통 안에서는 복원) |
| `Enter` / `Space` | 상세 뷰어 열기 (다중 선택 모드에서는 선택 토글) |
| `←` / `→` / `↑` / `↓` 또는 `h j k l` | 포커스 이동 |
| `Esc` | 포커스 / 상세 뷰어 닫기 |
| `p` | 비교 핀 토글 |

포커스 이동 시 푸른색 링(4px ring-blue-500)으로 현재 선택된 카드가 강조되며, 자동으로 화면에 스크롤됩니다.

### 다중 선택 모드

여러 장을 한꺼번에 처리하려면 다중 선택 모드를 사용합니다.

#### 선택 방법

1. 도구막대의 **선택 모드** 버튼을 켭니다.
2. 원하는 이미지를 클릭해 선택합니다.
3. **Shift+클릭**으로 범위 선택 또는 누적 선택.
4. **드래그 선택 (마퀴)**: 그리드 영역에서 마우스를 누른 채 드래그하면 사각형 선택 영역이 그려지고, 해당 영역과 겹치는 모든 카드가 선택됩니다. 드래그 중 스크롤해도 정확한 교차 계산이 유지됩니다.
5. **장시간 터치** (모바일): 이미지에 길게 누르면 선택 모드가 켜지고 해당 이미지가 선택됩니다.

#### 선택 모드 액션 바

선택 모드가 활성화되면 상단에 푸른색 액션 바가 표시됩니다:

- 선택된 이미지 수가 표시됩니다.
- **일괄 통과** (`1` 키): 선택된 모든 이미지를 통과 상태로 변경
- **일괄 탈락** (`2` 키): 선택된 모든 이미지를 탈락 상태로 변경
- **일괄 대기** (`3` 키): 선택된 모든 이미지를 대기 상태로 변경
- **일괄 휴지통**: 선택된 모든 이미지를 휴지통으로 이동
- **일괄 다운로드**: 선택된 이미지를 ZIP 파일로 다운로드 (`gallery-images.zip`)
- **선택 종료** (`Esc` 키): 선택 모드 해제

일괄 작업 결과는 토스트 메시지와 액션 바에 표시됩니다.

#### 전체 선택/해제

페이지네이션 영역에 **전체 선택 / 전체 해제** 버튼이 있습니다. 현재 페이지의 모든 이미지를 한 번에 선택하거나 해제합니다.

## 4.6 상세 뷰어 (ImageDetail)

이미지 카드를 클릭하면 상세 뷰어가 모달로 열립니다.

- **이미지**: 원본 크기 비율로 표시 (최대 60vh 높이)
- **상태 배지**: 큐레이션 상태 표시
- **파일명**: 모노스페이스 폰트로 표시
- **다운로드 버튼**: 새 탭 또는 직접 다운로드 (설정 따라 다름)
- **메타데이터**: 해시, 프롬프트, 파일 크기, 워커 ID
- **워크플로우 저장**: 이미지에 연결된 ComfyUI 워크플로우를 이름과 함께 저장 (워크플로우 관리 탭에서 재사용 가능)
- **템플릿 저장**: CEG 템플릿을 이름과 함께 저장 (템플릿 관리 탭에서 재사용 가능)
- **노트**: 자유 형식 메모 작성 및 저장
- **태그 관리**:
  - 기존 태그를 클릭하여 삭제 (툴팁: \"태그 삭제\")
  - 새 태그 입력 및 추가 (Enter 키 또는 \"추가\" 버튼)

저장 버튼이 있는 필드(노트, 워크플로우, 템플릿)는 명시적으로 저장해야 반영됩니다.

## 4.7 ImageViewer (전체 화면 이미지 뷰어)

비교 뷰의 마그니파이어와 함께 사용되는 전문 이미지 뷰어입니다.

### 확대/축소

- **마우스 휠**: 커서 중심을 기준으로 확대/축소 (1x~6x)
- **좌클릭**: 한 단계 확대 (1x → 1.5x → 2x → 3x → 4x → 6x)
- **우클릭**: 한 단계 축소
- 확대 단계는 고정된 스텝(1, 1.5, 2, 3, 4, 6)을 따릅니다.

### 이동 (팬)

- **드래그**: 이미지를 드래그하여 이동 (확대 배율과 관계없이 항상 사용 가능)
- 커서는 드래그 중 `grabbing`, 평소 `grab`으로 변경됩니다.

### 영역 확대 (Shift+드래그)

- **Shift 키 + 드래그**: 사각형 선택 영역을 그립니다.
- 선택 영역을 놓으면 해당 영역이 화면에 맞게 자동 확대됩니다.
- 선택 영역이 너무 작으면(4px 미만) 일반 클릭으로 처리됩니다.
- 커서는 `crosshair`로 변경됩니다.

### 돋보기 렌즈

마우스를 올리면 커서 근처에 **부동 돋보기 렌즈**가 표시됩니다.

- 렌즈 크기: 80px~300px (설정에서 조절)
- 렌즈 확대 배율: 1.5x~5x (설정에서 조절)
- 렌즈 모양: 원형 / 사각형 토글
- 렌즈 ON/OFF 토글 버튼
- 설정 패널에서 크기, 확대 배율, 모양을 슬라이더와 버튼으로 조절 가능

### 기타

- **Esc**: 뷰어 닫기
- **Shift**: 키보드 Shift 키를 누르면 Shift 모드가 활성화됨
- **로딩 상태**: 스핀너 애니메이션 표시
- **에러 상태**: 이미지 로딩 실패 시 아이콘 + 메시지 표시
- 화면 크기에 맞춰 자동 크기 조정 (최대 92vw × 92vh)

## 4.8 태그 폴더 정리 (브레드크럼)

이미지를 태그 기반으로 가상 폴더로 묶어 정리할 수 있습니다.

- 그리드 상단의 **브레드크럼**으로 현재 위치를 확인하고, 상위 폴더로 이동합니다.
- 하위 폴더 버튼을 클릭하여 태그로 추가 필터링합니다.
- 폴더 검색 입력창으로 하위 태그를 빠르게 찾습니다.
- 우클릭 메뉴의 **하위 폴더로 이동** 서브 메뉴로 빠르게 이동합니다.
- 빈 영역 우클릭 → **상위 폴더로 이동** / **홈 폴더로 이동**

브레드크럼 태그 시스템은 백엔드의 실제 폴더 구조와 무관하며, 이미지 파일명과 태그에서 추출된 토큰을 기반으로 가상 폴더를 생성합니다.

## 4.9 노트와 태그 편집

상세 뷰어에서 다음을 입력할 수 있습니다.

- **태그** — 개별 태그 추가/제거. 기존 태그를 클릭하면 삭제됩니다.
- **노트** — 자유 형식의 메모입니다. \"노트 저장\" 버튼으로 명시적 저장.
- **워크플로우 저장** — 이미지의 ComfyUI 워크플로우를 이름과 함께 저장합니다.
- **템플릿 저장** — CEG 템플릿을 이름과 함께 저장합니다.

## 4.10 휴지통

휴지통은 \"지울지 한 번 더 생각하는\" 안전망입니다.

- `3` 키나 우클릭 → 휴지통으로 이미지를 보냅니다.
- 상태 필터를 **휴지통**으로 바꾸면 휴지통 안의 이미지가 보입니다.
- 휴지통 안에서 `3`을 누르면 다시 대기(pending) 상태로 복원됩니다.
- 카드 오른쪽 위의 휴지통 버튼으로 빠르게 휴지통/복원 토글.
- 도구막대의 **휴지통 비우기**(더보기 메뉴 안)로 영구 삭제합니다. 확인 대화상자가 한 번 뜨고, 삭제된 개수가 토스트로 표시됩니다.

## 4.11 데이터셋 익스포트

훈련용 데이터셋을 만들 때 사용합니다. 설정에서 익스포트 범위를 구성할 수 있습니다:

- **익스포트 범위**: \"통과된 이미지만\" 또는 \"전체\"
- **중복 처리 전략**: \"해시 추가\" 또는 \"번호 추가\"

백엔드 API를 통해 ZIP 파일로 다운로드되며, 이미지와 함께 메타데이터(태그·노트·축 값)가 같이 들어갑니다.

용량이 크면 시간이 걸릴 수 있습니다. 완료 시 토스트 메시지가 표시됩니다.

## 4.12 실시간 갱신 (WebSocket)

갤러리는 백엔드의 WebSocket 이벤트를 구독하여 **실시간 자동 갱신**됩니다.

- `image.saved`: 새 이미지 저장 시
- `image.curation`: 큐레이션 상태 변경 시
- `image.deleted`: 이미지 삭제 시

이벤트 수신 시 자동으로 목록을 다시 가져옵니다. 연결이 끊기면 자동 재연결됩니다.

## 4.13 페이지네이션

이미지 수가 페이지 크기를 초과하면 페이지네이션 바가 표시됩니다.

- **그리드 모드**: 기본 페이지당 48개 (24/48/96으로 설정 가능)
- **그룹 모드**: 페이지당 20개 그룹
- 현재 페이지 ±1 페이지 + 첫/마지막 페이지 표시, 나머지는 생략부호(…)
- 이전/다음 버튼, 페이지 번호 직접 클릭
- 총 개수와 현재 페이지 정보가 표시됩니다.
- 필터 변경 시 자동으로 1페이지로 돌아갑니다.
- 전체 페이지 수가 줄어들어 현재 페이지가 범위를 벗어나면 자동으로 마지막 페이지로 조정됩니다.

## 4.14 그 외 도구

- **새로고침** — `Ctrl+Shift+R` 또는 더보기 메뉴의 새로고침 버튼으로 백엔드에서 다시 목록을 가져옵니다.
- **재생성** — 그룹 모드에서 그룹 헤더의 \"재생성\" 버튼으로 동일 그룹에 새 이미지 생성 요청. 수량 입력 다이얼로그 (1~100).
- **레이저 로딩** — 이미지 로딩은 `loading="lazy"` 속성으로 지연 로딩됩니다. 성능 최적화를 위해 뷰포트에 진입한 이미지만 로딩합니다.
- **유동 그리드 레이아웃** — CSS Grid `auto-fill` + `minmax`로 화면 크기에 맞춰 열 수를 자동 조절합니다.

## 4.15 단축키 (갤러리)

| 키 | 동작 |
|---|---|
| `←` / `h` | 이전 이미지 |
| `→` / `l` | 다음 이미지 |
| `↑` / `k` | 위쪽 이미지 (4칸 뒤로) |
| `↓` / `j` | 아래쪽 이미지 (4칸 앞으로) |
| `Enter` / `Space` | 상세 뷰어 열기 / 선택 토글 |
| `1` | 통과 |
| `2` | 탈락 |
| `3` | 휴지통 이동 / 복원 |
| `p` | 비교 핀 토글 |
| `Esc` | 포커스 해제 / 상세 뷰어 닫기 / 선택 모드 종료 |
| `Ctrl+Shift+R` | 목록 새로고침 |

전체 단축키는 화면 어디서나 `?` 키로 띄울 수 있습니다.
